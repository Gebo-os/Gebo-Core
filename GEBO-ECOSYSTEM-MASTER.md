## 1. Owner Node

The Owner Node is the **command center and source of authority**.

It controls:

- identity
- permissions
- memory
- routing
- local models
- external model access
- tool access
- Presence activation
- system rules
- user approval
- native Gebo infrastructure

**No external action without approval.**

Owner Node must eventually become Gebo-owned and **not** depend permanently on OpenAI, Claude, Obsidian, N8N, PostgreSQL, Supabase, or any public tool.

Temporary tools may be used only to **learn, prototype, build, and replace**.

**In this repo:** `gebo-core-private` is the Owner NODE V0 shell — local FastAPI + SQLite + Ollama + approval gate.
